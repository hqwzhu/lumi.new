# Lumi OS License Activation

This document explains the Windows MVP authorization flow for operators and support staff.

## User Flow

1. User installs Lumi OS.
2. First launch shows `Lumi OS 授权激活`.
3. User copies the machine code.
4. Operator opens `https://www.enhe-tech.com.cn/admin/license-generator`.
5. Operator generates a license code for that machine code.
6. User pastes the license code into Lumi OS and clicks `激活并继续`.
7. Lumi OS enters the API Key setup wizard.

## License Format

Lumi OS verifies this offline format:

```text
LUMI1-<base64url-json-payload>.<base64url-ed25519-signature>
```

Payload example:

```json
{
  "v": 1,
  "product": "lumi-os",
  "machineCode": "LUMI-WIN-ABCDE12345",
  "licenseId": "LIC-20260624-001",
  "issuedAt": "2026-06-24T00:00:00.000Z",
  "expiresAt": "2027-06-24T00:00:00.000Z",
  "edition": "windows"
}
```

Required fields: `v`, `product`, `machineCode`, `licenseId`, `issuedAt`.

Optional fields: `expiresAt`, `edition`.

## Generator Sync Requirement

The admin website generator must:

1. Normalize the machine code to uppercase and hyphen separators.
2. Build the JSON payload.
3. Convert the exact JSON string to base64url.
4. Sign the payload segment bytes with Ed25519 private key.
5. Return `LUMI1-${payloadSegment}.${signatureSegment}`.

The private key must only live on the admin generator or operator machine. Do not commit it to GitHub and do not include it in public download packages.

## Local Operator Utility

Initialize a key pair:

```powershell
node scripts/generate-license-code.mjs --init-keypair --out-dir <secure-license-authority-dir>
```

Generate a license:

```powershell
node scripts/generate-license-code.mjs `
  --machine-code LUMI-WIN-ABCDE12345 `
  --private-key-file <secure-license-authority-dir>\lumi-license-private-key.pem `
  --license-id LIC-20260624-001
```

The generated JSON output contains `licenseCode`; send only that code to the user.

## User Troubleshooting

- `Authorization code must start with LUMI1-`: the pasted code is incomplete.
- `Authorization signature does not match`: the code was not generated by the Lumi OS license generator.
- `Authorization code belongs to another computer`: the code was generated for a different machine code.
- `Authorization code has expired`: generate a new license code.

## 中文说明

Windows MVP 版本采用“一机一授权”：

1. 用户首次启动 Lumi OS 后复制机器码。
2. 运营人员在授权码生成器里输入机器码。
3. 生成授权码后发给用户。
4. 用户粘贴授权码并激活。
5. 激活成功后进入 API Key 配置向导。

私钥只能放在后台授权码生成器或运营人员安全电脑上，不能放进用户安装包，也不能提交到 GitHub。
